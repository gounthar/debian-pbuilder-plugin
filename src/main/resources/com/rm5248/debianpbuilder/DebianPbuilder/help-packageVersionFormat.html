The format of the package name. This determines what the version of the package will be when building a non-tag release.

Acceptable format specifiers:

  • YYYY - 4 digit year
  • MM - 2 digit month
  • dd - 2 digit day
  • HH - 2 digit hour
  • mm - 2 digit minute
  • ss - 2 digit seconds
  • %rev% - Revision. For git, this is a string like "git1234567"(contains last commit). For SVN, this is a string like "svn1234"(contains current rev)
  • %build% - Build number, as reported by Jenkins
The default value is YYYYMMddHHmmss.%rev%.%build%
